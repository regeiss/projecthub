Phase 1 Design — Section 2: Backend API

Two backend changes: serializer ancestors field + updated consumer

WikiPageDetailSerializer — add ancestors
class WikiPageDetailSerializer(serializers.ModelSerializer):
    ancestors = serializers.SerializerMethodField()

    def get_ancestors(self, obj):
        """Walk parent chain, return ordered list root→parent."""
        chain, page = [], obj.parent
        while page:
            chain.append({"id": str(page.id), "title": page.title})
            page = page.parent
        return list(reversed(chain))
        # e.g. [{"id":"..","title":"Engineering"},
        #        {"id":"..","title":"Backend"}]
Performance note: ancestors query walks up the parent chain with one DB hit per level. Typical wiki depth is 2–4 levels — acceptable. For deep trees (>10 levels), can add select_related chain or switch to django-mptt later without API change.
