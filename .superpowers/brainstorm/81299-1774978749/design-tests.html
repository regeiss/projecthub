Phase 1 Design — Section 4: Tests

Wiki currently has zero tests. Phase 1 establishes the full test foundation.

Backend tests — Django TestCase (apps/wiki/tests/)
test_models.py
  • WikiSpace creation (global + project-scoped)
  • WikiPage parent/child hierarchy
  • sort_order midpoint logic
  • WikiPageVersion auto-increment
  • WikiIssueLink link types
test_views.py
  • Space CRUD (list, create, update, delete)
  • Page CRUD + move + publish
  • Page detail includes ancestors field
  • Version list + restore
  • Comment create, resolve, delete
  • Private space access control
  • Locked page edit blocked
  • Public page (no auth) endpoint
test_consumers.py
  • Connect sends TipTap JSON init message
  • Connect sends Yjs binary if yjs_state set
  • Binary message relayed to group (not sender)
  • Archived page connect rejected
  • Unauthenticated connect rejected
test_tasks.py
  • save_yjs_state writes to yjs_state field
  • save_yjs_state does not touch content field
  • create_page_version snapshots title + content
  • version_number auto-increments correctly
Frontend tests — Vitest + React Testing Library (features/wiki/__tests__/)
WikiBreadcrumb.test.tsx
  • Renders space + ancestor links
  • Current page not a link
  • Empty ancestors = space only
  • Each segment navigates correctly
WikiTOC.test.tsx
  • Renders headings from TipTap JSON
  • H2 indented under H1
  • Hidden when <3 headings
  • Click calls scrollIntoView
useWiki.test.ts
  • useWikiPage returns ancestors array
  • useUpdateWikiPage sends TipTap JSON
  • useWikiSpaces filters by workspace
Scope note: WikiEditor.tsx (TipTap + Yjs) is excluded from unit tests — too many browser APIs to mock reliably. Covered by integration tests in a future phase when Playwright is added.
