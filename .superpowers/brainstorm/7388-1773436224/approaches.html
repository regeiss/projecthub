Dark Mode — Implementation Approaches

Three ways to add dark mode to a Tailwind + React app. Pick the one that fits best.

A

Tailwind dark: variant + class strategy

Add darkMode: 'class' to Tailwind config. Toggle a dark class on <html>. Every component uses dark:bg-gray-900 style utilities. Store preference in localStorage + a small React context.

Pros

  • Native Tailwind — no extra libs
  • Full control over each token
  • No runtime style injection

Cons

  • Every component needs dark: variants added
  • More lines of code across the codebase

★ Recommended

B

CSS custom properties (variables)

Define a design token layer in CSS (--color-bg, --color-text, etc.) with :root and [data-theme="dark"] blocks. Components use bg-[var(--color-bg)] or a custom Tailwind theme. One class flip on <html> switches everything.

Pros

  • Single source of truth for color tokens
  • Easy to add more themes later
  • Clean component code

Cons

  • Requires refactoring all color classes to variables
  • Higher upfront investment
C

External library (next-themes / use-dark-mode)

Use a library like next-themes (works without Next.js) or use-dark-mode to manage theme state, system preference detection, and localStorage persistence automatically.

Pros

  • Handles edge cases (SSR flash, system preference)
  • Minimal boilerplate

Cons

  • Adds a dependency
  • Still need Tailwind dark: variants on components
  • Overkill for a pure Vite/React app
